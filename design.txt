----
ДАНО

Автовокзал осуществляет продажу билетов на пригородные рейсы.
Для каждого рейса указаны:
-   номер рейса;
-   тип автобуса;
-   пункт назначения;
-   дата отправления;
-   время отправления;
-   время прибытия на конечный пункт;
-   стоимость одного билета;
-   количество оставшихся для продажи билетов;
-   количество проданных билетов.

Общее для всех вариантов задание:
-   реализовать авторизацию для входа в систему;
-   функционал администратора и функционал пользователя.

Функционал пользователя включает:
    просмотр, поиск и сортировку данных структур.
Функционал администратора включает:
    просмотр, поиск и сортировку данных структур, добавление, редактирование, удаление.

Индивидуальное задание: обеспечить функциональную возможность покупки билетов
на конкретный рейс в требуемом количестве (при этом количество
оставшихся для продажи билетов и количество проданных билетов обновляется).

РЕАЛИЗАЦИЯ

Задание 1: авторизация, функционалы администратора и пользователя.

Проектирование (процедурный подход):
-   Данные:
        1.  Account[] accounts;
            1.1.    int n_accounts; // number of accounts
        2.  BusRoute[] routes;
            2.1.    int n_routes; // number of routes
        3.  Account current_account;
-   Этапы:
        1.  preparation():
            1.1.    generate_accounts();
            1.2.    write_accounts();
            1.3.    generate_routes();
            1.4.    write_routes();
            1.5.    read_accounts();
            1.6.    read_routes();
        2.  account_enter(); // menu
                2.1.    account_authorization();
                2.2.    account_registration();
                2.3.    quit();
        3.  account_actions();
                3.1.    user_actions(); // menu
                3.2.    admin_actions(); // menu
